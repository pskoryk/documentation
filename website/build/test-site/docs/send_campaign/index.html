Send campaign

POST Send Push Notification V3

Definition
Method: POST
Endpoint:https://uapi.gravitec.net/api/v3/push
Headers: Content-Type: application/json

Request body properties

  "send\_date": "",
  "ttl": "",
  "push\_tag": "",
  "display\_time": "",
  "is\_transactional": "",
  "ignore\_duplicate": " ",
  "segments": \[
    ""
  \],
  "payload": {
    "message": "",
    "title": "",
    "icon": "",
    "image": "",
    "redirect\_url": "",
    "buttons": \[
      {
        "title": "",
        "url": ""
      }
    \]
  },
  "audience": {
    "tokens": \[
      ""
    \],
    "aliases": \[
      ""
    \],
    "tags": \[
      ""
    \],
    "tags\_segment": \[
      {
        "association": "",
        "name": ""
      }
    \]
  }
}

Properties

Name

Type

Description

send_date
Optional
default: ""

String or Int

Schedule a campaign or send it immediately.

In order to schedule a campaign the desired date and time may be set, which have to be in the future.

For immediate sending set it to "now" or "".

NOTE 1

Time must be inUNIX_TIMEformat, which is essentially
the number of seconds that have elapsed since 00:00:00 Coordinated Universal Time (UTC), Thursday, 1 January 1970, minus the number of leap seconds that have taken place since then

Example

"send_date": "" - immediate sending
"send_date": 1509058800 – send push at 2017.Oct.26 23:00:00

ttl
Optional
default: 28800

Int

The maximum period of time for which Gravitec tries to deliver all the messages from a campaign

NOTE 1

The number must be in between 0 and 2 419 200

NOTE 2

Keep in mind that a TTL value of 0 means messages that can't be delivered immediately to users are discarded. However, because such messages are never stored, this provides the best latency for sending notification messages

Example

"ttl ": 28800 - time-to-live is set to 8 hours

push_tag

Optional

String

Tags push campaign so that a comparative analysis among such tags may be available.

NOTE 2

A tag's length must be between 1 and 100 characters

Example

"push_tag": "news" - tags a campaign with "news" tag

is_transactional
Optional
default: false

boolean

Marks a push message as Transactional, thus the message will not be displayed in the main campaign history section of the web application ({+}https://push.gravitec.net+).

Use this if you send a great deal of small messages, which are by itself contains transactional data like personal notifications for your users, reminders, etc.

NOTE 1

If omitted, "false" value will be set for a push message

ignore_duplicate

Optional

default: false

boolean

true: the system shall compare provided via the REST call push campaign data with all campaigns within a 24h inteval. If the system found a dublicate push campaing, It will return an error code and campaign won't be send

false: the system shall not run a search for dublicate push campaigns

segments

****Optional****

Array

Send push campaign to a segment.

NOTE 1

You can not include both the segments and the **audience **section in request.

NOTE 2

You can only send to one segment. In case more than one segment included the system will not send the campaign

NOTE 3

If segment has been deleted or does not exists the system will not send the campaign

payload
Required

The "Payload" section is required, whereas some arguments within the section are not required.

message
Required

String

Text, which will be displayed as push message text.

NOTE 1

Maximum length of the message is 1000 symbols.

Example

"push_tag": "news" - tags a campaign with "news" tag

title
Optional
default: Title

String

Title or header of the message

Chrome

Firefox

NOTE 1

Max length of title is 1000 symbols

Example

"header ": "Just another header"

icon
Required

String

Notification's icon URL. Sets the notification icon to specified image URL.

Image must be at least 80x80 pixels and with 1:1 ratio. Pictures without that ratio will be resized with its aspect ratio and put on a white square.

1:1 aspect ratio and more than 80x80 on X and Y sides:

1:1.2 aspect ratio and more than 80 on Y side:

1.2:1 aspect ratio and more than 80 on X side:

Less than 80x80 on X and Y sides:

NOTE 1

Provide correct URL, without spaces or tabs at the start and at the end of the URL and with ending .jpg, .jpeg or .png, otherwise the error will be fired

NOTE 2

Maximum length of the URL is 1000 symbols.

NOTE 3

Providing "icon": "" will set the icon to default site icon, which can be set in the Site Settings page

Example

"icon": "https://cdn.gravitec.net/img/gravitecBig.jpg"

image
Optional

String

URL of big picture to display below the notification main body.

For the best results provided image should be with 1.5:1 sides ratio and size at least 360x240 pixels.

Some image processing examples

1:1.2 aspect ratio and more than 240 on Y side:

1.2:1 aspect ratio and more than 360 on X side:


Less than 360x240 on X and Y sides:

1.5:1 aspect ratio and more or equal than 360x240 on X and Y sides accordingly:

NOTE 1

Provide valid URL without spaces or tabs at the start and at the end of the URL and with ending with .jpg, .jpeg or .png, otherwise the error will be fired

NOTE 2

Maximum length of the URL is 1000 symbols.

NOTE 3

Pictures, which are not in 1.5:1 ratio or less then 360x240 and will be resized according to their aspect ratio and put on top of a white image of 360x240 px

redirect_url
Required

String

URL which will be opened in user's browser if user clicks on the notification body.

NOTE 1

Maximum length of the URL is 1000 symbols.

NOTE 2

Provide correct URL, without spaces or tabs at the start and at the end of the URL, otherwise the error will be fired.

Example

"redirect_url": "http://yoursite.com/news"

buttons
Optional

Specify in this array one or two action buttons, which will be displayed below the notification main body.

title
Required

String

A button title

NOTE 1

Max length of title is 48 symbols

url
Required

String

URL which will be opened in user's browser if user clicks on the button.

NOTE 1

Maximum length of the URL is 1000 symbols.

NOTE 2

Provide correct URL, without spaces or tabs at the start and at the end of the URL (rfc3986)

audience
Optional

The "Audience" section is optional.

NOTE 1

If omitted, the system shall send push message to all subscribers.

NOTE 2

If set - the section must contain only one of these variables (in other case, error will be fired):

tokens
Optional

Array

A list of subscribers' IDs in this parameter means that the campaign will be send exactly to the list of subscribers specified.

You can get subscriber's ID by usinggetSubscriptionfunction from ourWeb SDK API

NOTE 1

Array size is limited to 100 IDs

Example

"tokens": ["dec301908b9ba...8df85e57a58e40f96f", "523f4c2068674f1fe...2ba25cdc250a2a41"]

aliases
Optional

Array

A list of aliases means that the campaign will be sent exactly to the list of subscribers who are associated with aliases provided in the list.

You can set an alias by using setAlias function from ourWeb SDK API

Example

"aliases": ["own_id_device_1", "own_id_device_2", "own_id_device_3"]

tags
Optional

Array

A list of tags means that message will be send exactly to the list of subscribers who were tagged with tags provided in the list.

You can tag your subscribers by using addTag or setTags function from ourWeb SDK API

Example

"tags": ["tag1", "tag2", "tag3"]

tags_segment

Optional

Array

Target subscribers by filtering them with tags conditions

You can tag your subscribers by using addTag or setTags function from ourWeb SDK API

In order to send campaign to a part of your audience that is an intersection of two or more segments of tagged subscribers you can provide such a condition by using logic operators AND, AND_NOT, OR, OR_NOT followed by a tag. Such conditions should be put in array of json elements.
AND – include all followers tagged by the tag followed with AND logical operator
AND_NOT - include all followers who hasn't been tagged by the tag followed with AND logical operator
OR - include all followers tagged by the tag followed with OR logical operator
OR_NOT - include all followers who hasn't been tagged by the tag followed with OR logical operator

NOTE 1

Tagsarray must contain not less that 1 and not more than 256 elements

NOTE 1

Tag name must contain not less that 1 and not more than 255 symbols

NOTE 1

You must provide logical operator only from the list: AND, AND_NOT, OR, OR_NOT

NOTE 1

Tag name provided must be case-insensitive unique among the elements in the array (Example: CARS and cars are not unique)

NOTE 1

It is not allowed to use both tags_segment and tags directive simultaneously

NOTE 1

You must provide tags that exists in the system

Example

"tags_segment": [
{
"association": "AND",
"name": "tag1"
} ,
{
"association": "AND_NOT",
"name": "tag2"
}
]
The system shall send the campaign to all followers with tag tag1 and without tag2

Example

"tags_segment": [
{
"association": "AND_NOT",
"name": "tag1"
} ,
{
"association": "OR",
"name": "tag2"
}
]
The system shall send the campaign to all followers without tagtag1or withtag2
